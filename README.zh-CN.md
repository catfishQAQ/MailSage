# MailSage

[English](README.md) | [简体中文](README.zh-CN.md)

用本地 AI 总结邮件，不把收件箱数据发送到云端。

MailSage 是一个强调隐私与本地处理的邮件助手，适合希望更快整理收件箱、同时又不想把邮件内容发送到云端大模型的用户。

它可以把多个邮箱集中到一个应用中管理，并通过 [Ollama](https://ollama.com/) 调用本地模型，对邮件进行总结、提炼重点并辅助生成回复草稿，整个 AI 处理流程尽量留在你的电脑上完成。

## 应用特点

- 多邮箱集中管理：一个应用里接入并管理多个邮箱账号。
- 基于 Ollama 的本地 AI：不依赖云端 LLM API，在本机完成邮件总结和回复辅助。
- 更重视隐私：邮件分析过程留在本地，能更好降低数据外发风险。
- 提高处理效率：自动生成摘要、行动项和回复草稿，帮助你更快清理收件箱。
- 配置更省事：内置常见邮箱服务商的服务器预设，接入更快捷。

## 当前能力范围

- 多账号邮件接入，当前基于 IMAP 同步
- 基于 Ollama 的本地邮件总结与回复辅助
- 支持英文和简体中文界面

## 本地模型建议

请根据自己机器的显存和内存大小下载不同的 Ollama 模型进行适配，这样应用才能在你的硬件上更稳定地运行。

示例：

- RTX 2060 6GB：`qwen3:4b-q4`
- RTX 3090 24GB：`qwen3.5:27b-q4`


根据他人的测试，Qwen3 及之后的模型中，参数规模在 4B 或以上的模型能够胜任每日邮件总结的任务。不过我还没有亲自测试过。如果你愿意，可以尝试使用更小的模型，看看效果如何，并将你的反馈分享给社区。总体来说，4B 以上的模型似乎是更好的选择。

## 一键启动

MailSage 现在支持“一次点击启动”的日常使用方式。

- 开发模式：继续使用原来的前后端分离方式，访问 `http://localhost:5173`
- 一键启动模式：Windows 双击 `MailSage.bat`，macOS / Linux 运行 `./start-mailsage.sh`，随后访问 `http://127.0.0.1:8000`

一键启动器会在需要时自动构建前端、启动负责托管前端页面的后端，并自动打开浏览器。

如果之后想关闭一键启动模式拉起的后端，Windows 可以双击 `Stop-MailSage.bat`，macOS / Linux 可以运行 `./stop-mailsage.sh`。
