# MailSage

[English](README.md) | [简体中文](README.zh-CN.md)

MailSage 是一个强调隐私与本地处理的邮件助手，适合希望更快整理收件箱、同时又不想把邮件内容发送到云端大模型的用户。

它可以把多个邮箱集中到一个应用中管理，并通过 [Ollama](https://ollama.com/) 调用本地模型，对邮件进行总结、提炼重点并辅助生成回复草稿，整个 AI 处理流程尽量留在你的电脑上完成。

## 应用特点

- 多邮箱集中管理：一个应用里接入并管理多个邮箱账号。
- 基于 Ollama 的本地 AI：不依赖云端 LLM API，在本机完成邮件总结和回复辅助。
- 更重视隐私：邮件分析过程留在本地，能更好降低数据外发风险。
- 提高处理效率：自动生成摘要、行动项和回复草稿，帮助你更快清理收件箱。
- 配置更省事：内置常见邮箱服务商的服务器预设，接入更快捷。

## 当前能力范围

- 多账号邮件接入，当前基于 IMAP 同步
- 基于 Ollama 的本地邮件总结与回复辅助
- 支持英文和简体中文界面

## 如何在 GitHub 放双语 README

GitHub 仓库首页默认只会自动展示一个 README，也就是 `README.md`。

如果你想做双语展示，推荐这样组织：

1. 把 `README.md` 作为主入口页。
2. 再新增一个中文文件，比如 `README.zh-CN.md`。
3. 在两个 README 顶部都放上语言切换链接。

示例：

```md
[English](README.md) | [简体中文](README.zh-CN.md)
```

这样用户在 GitHub 上点击语言链接时，就会跳转到对应语言版本的 README。
